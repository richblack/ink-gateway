## 一般問題

### Q1: Obsidian Ink Plugin 是什麼？

**A**: Obsidian Ink Plugin 是一個為 Obsidian 筆記軟體開發的插件，它將您的筆記與 Ink-Gateway 系統整合，提供 AI 聊天、語義搜尋、模板管理和自動內容同步等功能。透過這個插件，您可以：

- 與 AI 助手進行智能對話
- 使用語義搜尋快速找到相關內容
- 建立和使用可重複的內容模板
- 自動同步筆記到雲端資料庫
- 支援離線模式和衝突解決

### Q2: 使用這個插件需要付費嗎？

**A**: 插件本身是免費的開源軟體，但您需要：

- **Obsidian**: 個人使用免費，商業使用需要授權
- **Ink-Gateway 服務**: 根據您選擇的服務提供商而定
  - 自架設伺服器：免費（需要技術能力）
  - 託管服務：可能需要訂閱費用
- **AI 服務**: 根據使用量計費（如 OpenAI API）

### Q3: 這個插件與其他筆記軟體相容嗎？

**A**: 目前插件專為 Obsidian 設計，但 Ink-Gateway 系統支援多種筆記應用：

- **已支援**: Obsidian、Logseq、RemNote
- **計劃支援**: Notion、Roam Research、Dendron
- **資料格式**: 標準 Markdown，易於遷移

### Q4: 我的資料會儲存在哪裡？

**A**: 您的資料儲存在多個位置：

1. **本地**: Obsidian 庫中的原始檔案
2. **本地快取**: 插件快取資料（可清除）
3. **Ink-Gateway**: 您設定的伺服器位置
   - 自架設：您的伺服器
   - 託管服務：服務提供商的伺服器
4. **備份**: 建議定期備份 Obsidian 庫

## 安裝和設定

### Q5: 系統需求是什麼？

**A**: 
- **Obsidian**: 版本 1.0.0 或更高
- **作業系統**: Windows 10+、macOS 10.15+、Linux (Ubuntu 18.04+)
- **記憶體**: 建議 4GB 以上
- **儲存空間**: 至少 100MB 可用空間
- **網路**: 穩定的網際網路連線

### Q6: 如何安裝插件？

**A**: 有兩種安裝方式：

**方法一：社群插件商店（推薦）**
1. 開啟 Obsidian → 設定 → 社群插件
2. 搜尋 "Ink Plugin"
3. 點擊安裝並啟用

**方法二：手動安裝**
1. 從 [GitHub Releases](https://github.com/your-repo/obsidian-ink-plugin/releases) 下載
2. 解壓到 `.obsidian/plugins/obsidian-ink-plugin/`
3. 重新啟動 Obsidian 並啟用插件

### Q7: 如何設定 Ink-Gateway 連線？

**A**: 
1. 啟用插件後，前往 **設定 → Ink Plugin**
2. 填入以下資訊：
   - **Gateway URL**: 您的 Ink-Gateway 伺服器位址
   - **API Key**: 從 Gateway 管理介面獲取
3. 點擊 **測試連線** 驗證設定
4. 儲存設定並開始使用

### Q8: 我沒有 Ink-Gateway 伺服器，如何獲得？

**A**: 有幾種選擇：

**選項一：使用託管服務**
- 聯繫官方獲取託管服務資訊
- 通常包含設定協助和技術支援

**選項二：自架設伺服器**
- 下載 Ink-Gateway 開源版本
- 按照文件進行安裝和設定
- 需要一定的技術背景

**選項三：試用版本**
- 申請免費試用帳號
- 適合評估和測試使用

## 功能使用

### Q9: AI 聊天功能如何使用？

**A**: 
1. **開啟聊天視窗**：
   - 快捷鍵：`Ctrl+Shift+A`
   - 命令面板：搜尋 "Open AI Chat"
   - 側邊欄：點擊 AI 圖示

2. **開始對話**：
   - 直接輸入問題或指令
   - AI 會基於您的筆記內容回答
   - 支援多輪對話和上下文理解

3. **常用指令**：
   - "總結這份文件"
   - "幫我找相關的筆記"
   - "解釋這個概念"
   - "改善這段文字"

### Q10: 語義搜尋與普通搜尋有什麼不同？

**A**: 

| 特性 | 普通搜尋 | 語義搜尋 |
|------|----------|----------|
| 匹配方式 | 關鍵字完全匹配 | 理解語義和概念 |
| 搜尋範圍 | 檔案名稱和內容 | 所有同步的內容 |
| 結果相關性 | 基於關鍵字頻率 | 基於語義相似度 |
| 範例 | 搜尋 "機器學習" 只找到包含這個詞的內容 | 搜尋 "機器學習" 也會找到 "AI"、"神經網路" 等相關內容 |

### Q11: 如何建立和使用模板？

**A**: 
**建立模板**：
```markdown
# 會議記錄模板

**日期**: {{date}}
**參與者**: {{participants}}
**主題**: {{topic}}

## 議程
{{agenda}}

## 決議事項
{{decisions}}

#會議記錄 #{{project_tag}}
```

**使用模板**：
1. 建立新筆記或開啟現有筆記
2. 命令面板 → "Apply Template"
3. 選擇模板並填入插槽值
4. 確認套用

### Q12: 自動同步是如何運作的？

**A**: 
1. **觸發條件**：
   - 編輯內容後按 Enter
   - 檔案建立、刪除、重新命名
   - 定時同步（可設定間隔）

2. **同步過程**：
   - 解析內容結構和階層
   - 建立統一區塊格式
   - 上傳到 Ink-Gateway
   - 更新本地快取

3. **狀態指示**：
   - 🟢 已同步
   - 🟡 同步中
   - 🔴 同步失敗
   - ⚫ 離線模式

## 效能和穩定性

### Q13: 插件會影響 Obsidian 的效能嗎？

**A**: 插件經過效能最佳化，正常使用下影響很小：

**效能最佳化措施**：
- 智能快取機制
- 背景處理同步
- 延遲載入功能
- 記憶體使用監控

**如果遇到效能問題**：
- 調整同步間隔（增加到 10-30 秒）
- 減少快取大小
- 關閉不需要的功能
- 定期清理快取

### Q14: 如何處理大量筆記的同步？

**A**: 
**初次同步建議**：
1. 選擇網路狀況良好的時間
2. 暫時關閉其他耗網路的應用
3. 分批同步（每次 100-200 個檔案）
4. 監控同步進度和錯誤

**日常維護**：
- 設定合適的同步間隔
- 定期清理不需要的檔案
- 使用標籤篩選同步範圍
- 監控儲存空間使用

### Q15: 離線模式如何運作？

**A**: 
**自動切換**：
- 插件自動偵測網路狀態
- 網路中斷時自動進入離線模式
- 網路恢復時自動開始同步

**離線功能**：
- ✅ 正常編輯和建立筆記
- ✅ 本地搜尋功能
- ✅ 套用已快取的模板
- ✅ 查看 AI 聊天歷史
- ❌ AI 即時對話
- ❌ 語義搜尋新內容
- ❌ 即時同步

**同步恢復**：
- 變更會排隊等待同步
- 網路恢復後按順序處理
- 自動解決簡單衝突
- 複雜衝突需要手動處理

## 資料安全和隱私

### Q16: 我的資料安全嗎？

**A**: 我們採用多層安全措施：

**傳輸安全**：
- 使用 HTTPS/TLS 加密傳輸
- API Key 認證機制
- 請求簽名驗證

**儲存安全**：
- 資料儲存在您控制的伺服器
- 支援資料庫加密
- 定期安全更新

**隱私保護**：
- 不會收集個人敏感資訊
- 本地處理優先
- 透明的資料使用政策

### Q17: 可以加密我的筆記內容嗎？

**A**: 
**目前支援**：
- 傳輸層加密（HTTPS）
- API 認證保護
- 本地快取加密

**計劃功能**：
- 端到端加密
- 客戶端加密選項
- 零知識架構

**臨時方案**：
- 使用 Obsidian 的加密插件
- 敏感內容使用密碼保護
- 定期備份到加密儲存

### Q18: 如何備份我的資料？

**A**: 
**自動備份**：
- Obsidian 庫定期備份
- Ink-Gateway 資料庫備份
- 版本歷史記錄

**手動備份**：
```bash
# 備份 Obsidian 庫
cp -r /path/to/vault /backup/location

# 匯出插件設定
命令面板 → "Export Settings"

# 匯出聊天歷史
AI 聊天視窗 → 設定 → 匯出歷史
```

**雲端備份**：
- 使用 Git 版本控制
- 同步到雲端儲存服務
- 定期匯出重要資料

## 整合和相容性

### Q19: 與其他 Obsidian 插件相容嗎？

**A**: 大部分情況下相容，但需注意：

**已測試相容**：
- Dataview
- Calendar
- Templater
- Advanced Tables
- Excalidraw

**可能衝突**：
- 其他同步插件（如 Obsidian Sync）

**解決衝突**：
1. 停用衝突插件測試
2. 調整插件載入順序
3. 聯繫技術支援

### Q20: 可以與其他筆記應用同時使用嗎？

**A**: 可以，但需要注意：

**支援場景**：
- 不同裝置使用不同應用
- 不同專案使用不同工具
- 逐步遷移到 Obsidian

**注意事項**：
- 避免同時編輯相同內容
- 注意資料格式差異
- 定期檢查同步狀態

### Q21: 支援行動裝置嗐？

**A**: 
**目前狀況**：
- Obsidian Mobile 支援有限
- 部分功能可能無法使用
- 主要針對桌面版最佳化

**行動裝置建議**：
- 使用 Obsidian Mobile 的基本功能
- 複雜操作在桌面版進行
- 等待官方行動版支援

## 故障排除

### Q22: 插件無法載入怎麼辦？

**A**: 按順序嘗試以下方法：

1. **重新啟動 Obsidian**
2. **檢查插件檔案**：
   - 確認 `main.js` 存在
   - 檢查 `manifest.json` 格式
3. **查看錯誤訊息**：
   - 開啟開發者工具 (`Ctrl+Shift+I`)
   - 查看 Console 中的錯誤
4. **重新安裝插件**：
   - 停用並刪除插件
   - 重新下載安裝

### Q23: 同步一直失敗怎麼辦？

**A**: 
**檢查清單**：
- [ ] 網路連線正常
- [ ] Gateway URL 正確
- [ ] API Key 有效
- [ ] 防火牆設定
- [ ] 磁碟空間充足

**解決步驟**：
1. 測試 Gateway 連線
2. 重新生成 API Key
3. 清除本地快取
4. 強制重新同步
5. 聯繫技術支援

### Q24: AI 聊天沒有回應？

**A**: 
**可能原因**：
- AI 服務暫時不可用
- API 配額用盡
- 網路連線問題
- 請求格式錯誤

**解決方法**：
1. 檢查 AI 服務狀態
2. 確認 API 配額
3. 重新啟動聊天功能
4. 聯繫服務提供商

### Q25: 搜尋結果不準確？

**A**: 
**改善方法**：
1. **等待索引完成**：
   - 新內容需要時間建立索引
   - 檢查索引進度

2. **調整搜尋參數**：
   - 降低相似度門檻
   - 增加結果數量
   - 使用不同關鍵字

3. **重建索引**：
   ```
   命令面板 → "Rebuild Search Index"
   ```

4. **檢查內容品質**：
   - 確保內容有意義的文字
   - 避免過多符號和代碼
   - 使用適當的標籤
